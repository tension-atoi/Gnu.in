Decision memo (2026-07-01, GO): `tooling/architecture/2026-07-01-layer-shell-host-and-blob-decision.md`.

**Move:** extract a generic `LayerShellHost<S: ShellSurface>` from the sctk loop that bar/dock/osd/launcher/notification each re-implement. Host owns Wayland side-effects (registry/seat/output/shm/pool/LayerSurface/frame-clock/Pixmap draw); per-surface trait provides `config()`, `scene(state,screen)->compose_core::Scene`, `update(state,event)->Vec<Msg>`. It's the Elm loop; compose-core already owns Scene/reducer/motion.

- Resolves the two blockers `gnuin-shell-chrome` cited for deferring: (1) standardize on PixmapŌåÆArgb8888 (retire bar's direct-BGRA); (2) `delegate_*!` bind once to the **generic** `LayerShellHost<S>`, not concrete types.
- `gnuin-compose-host`'s menu `App` = first `ShellSurface` impl (most complex; proves the trait).
- Migration (strangler): land WIP first ŌåÆ extract ŌåÆ menu host ŌåÆ osd ŌåÆ notification ŌåÆ **bar+dock (deletes the membrane structurally)** ŌåÆ launcher ŌåÆ fold `tooling/registry-catalog` into `gnuin-asset-core`.

**Scaffold landed 2026-07-01** on branch `feat/layer-shell-host` (commit 82bfb25): new crate `engine/gnuin-layer-host` ŌĆö pure `ShellSurface` trait + `SurfaceConfig`/`HostEvent` (sctk-free, 5 unit tests green) and a feature-gated generic `LayerShellHost<S>` lifted from compose-host's `App`. `delegate_*!(@<S: ShellSurface> LayerShellHost<S>)` **compiles against sctk 0.19** (the deferred-host blocker is empirically solved); scene rasterizer injected via `Box<dyn FnMut(&Scene,&mut Pixmap)>` to avoid a compose-host crate cycle. clippy-clean default + `--features wayland`.

**First port proof landed** (commit 79d7918): `engine/gnuin-osd/src/surface.rs` ŌĆö `OsdSurface: ShellSurface` maps osd's `OsdState` + `scene_for_state` onto the trait unchanged (config/scene/update); 3 new tests + full osd suite green, both build modes. Osd already builds a real `compose_core::Scene` and rasterizes via `gnuin_compose_host::render::paint_scene_with_font` (osd already deps on compose-host, so injecting it is cycle-free). **Key gap found:** the scaffold host uses `blocking_dispatch` and only pumps Wayland events; osd (and bar/dock/notification) need **calloop + external sources** ŌĆö osd runs a Unix-socket IPC channel + a 1400ms auto-hide `Timer` as extra calloop sources on a fullscreen `Layer::Overlay`. **Host calloop capability landed** (commit after 79d7918): host now uses `calloop::EventLoop` + `WaylandSource` (not `blocking_dispatch`); trait gained `type External: Send`, `on_external`, `on_timer`, and a generic `start<E: Fn(External)+Clone+Send>(emit)` hook; `SurfaceConfig::tick_interval`. Surfaces can feed their own IPC/watcher messages + run a periodic timer. Pure tests + `--features wayland` + clippy green.

**Remaining for osd cutover (task 18):** (1) un-gate `OsdMessage`/`OsdEvent` *types* from the `serde` cfg (keep JSON parse serde-gated) so `type External = OsdMessage` works; (2) wrap state as `OsdHostState { osd: OsdState, hide_deadline: Option<Instant> }` for the auto-hide poll (`on_timer`); (3) move the IPC reader thread into `OsdSurface::start(emit)`; (4) thin `main.rs` ŌåÆ `LayerShellHost::run`, delete the ~450-line `App`. **Behavior decision (Operator chose):** preserve per-output targeting ŌĆö build multi-output support first. **Done** (commit 64339e6): host manages one *retargetable* layer surface (tracks outputs by name; `HostCommand::ShowOn(output)` creates/retargets via tear-down+recreate, `Hide` tears down), driven by `on_external`/`on_timer` returning `Vec<HostCommand>`. `SurfaceConfig` gained `lazy()` (on-demand, unmapped until ShowOn) + `click_through()` (empty input region). All green.

Lets notification take input only on card rects while gaps pass through.

**Next = the notification SURFACE (task 20), host is ready, not yet built.** Shape: `NotificationHostState { notif: NotificationState (VecDeque stack, MAX_VISIBLE=4), deadlines: HashMap<popup_id,Instant>, width, height }`; External = pure `NotifInput` (Enqueue/Dismiss/DismissAll/Inhibit) mapped from serde-gated `NotificationMessage` in `start()`; on_external folds + arms per-popup `duration_ms` deadlines ŌåÆ `[ShowOn(screen), Redraw]` or `[Hide]` when empty; on_timer expires due popups (dismiss each), Hide if empty else Redraw; `update(PointerButton{x,y})` hit-tests by rebuilding `layout(state)` (needs stored width/height) ŌåÆ dismiss/activate the hit card; `input_region` = the card rects from `layout`; config: Layer::Overlay, anchor TOP|RIGHT|BOTTOM, size (NOTIF_COLUMN_WIDTH,0), keyboard None, `.lazy()`, `.tick(1s)`, NOT click_through. It's the most complex surface (interactive + per-item timers) and untestable in sandbox ŌĆö runtime-verify Operator-side.

**Then `gnuin-bar` + `gnuin-dock` ŌĆö whose move onto `compose_core::Scene` + this host structurally deletes the membrane** (single-owner NodeId::Bar). Always-visible (eager), interactive, external sources (battery/net/hypr) via `start(emit)`.

Compile-verifiable here; runtime-verify Operator-side. `paint_scene` relocation only matters for the *menu* host, not osd.
